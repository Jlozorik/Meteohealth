# 3.5 Тестирование

## Подход

Тестирование охватывает доменный слой (чистая логика без Android-зависимостей) через юнит-тесты. Инструментальные тесты не рассматриваются как приоритетные для данного проекта.

## Инструменты

| Инструмент | Назначение |
|---|---|
| JUnit 4 | Фреймворк запуска тестов |
| MockK 1.14.2 | Мок-объекты для Kotlin (suspend-функции, Flow) |
| Turbine 1.2.0 | Тестирование холодных и горячих Flow |
| kotlinx.coroutines-test | TestCoroutineDispatcher, runTest |

## Тестируемые компоненты

### WellbeingCalculator

Тесты проверяют корректность формулы индекса самочувствия:

- Базовый случай: нулевые отклонения → индекс 100.
- Граничные случаи: максимальные штрафы по каждому фактору → индекс не ниже 0.
- Персональные штрафы: гипертония при сильном перепаде давления, мигрени при высоком Kp.
- Накопление штрафов: несколько факторов одновременно.
- Результат всегда в диапазоне [0, 100].

### TriggerAnalyzer

Тесты проверяют корреляционный анализ:

- Возвращает пустой список при менее 14 записей.
- Идеальная прямая корреляция (wellbeing ∝ pressure) → r ≈ 1.0.
- Несвязанные данные → |r| < 0.3.
- Фактор исключается из результатов, если менее 5 записей содержат его значение.

## Структура тестов

```
app/src/test/java/com/meteohealth/
├── ExampleUnitTest.kt          — заглушка от Android Studio
└── domain/
    ├── WellbeingCalculatorTest.kt
    └── TriggerAnalyzerTest.kt
```

## Запуск

```bash
./gradlew :app:testDebugUnitTest
```

Отчёт: `app/build/reports/tests/testDebugUnitTest/index.html`

## Покрытие

Доменная логика (`WellbeingCalculator`, `TriggerAnalyzer`) покрыта тестами на граничные и типовые случаи. Репозитории и ViewModel не мокируются: их корректность верифицируется интеграционным запуском приложения с `FakeWeatherRepository` в DEBUG-режиме.
