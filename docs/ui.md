# 2.3 Пользовательский интерфейс

## Технологии

UI полностью построен на **Jetpack Compose** с библиотекой компонентов **Material 3**. Навигация — `Navigation Compose`. Графики — **Vico 2.1.2**.

## Тема

Приложение поддерживает тёмную и светлую темы без динамических цветов Material You. Выбор темы сохраняется в `user_profile.isDarkTheme` и применяется в `MeteohealthTheme(darkTheme = isDarkTheme)`. По умолчанию — тёмная.

**Тёмная тема (Midnight Sky):**

| Роль | Цвет |
|---|---|
| primary | #5BB9FF |
| background | #07091A |
| surface | #0D1023 |
| surfaceVariant | #141829 |
| onSurface / onBackground | #D8DFFF |
| secondary | #FFAB40 |
| tertiary | #5EEAD4 |
| error | #FF6B6B |

Дополнительные цвета для индекса благополучия: `WellbeingGood` (#5BB9FF), `WellbeingModerate` (#FFAB40), `WellbeingPoor` (#FF7043), `WellbeingDanger` (#FF6B6B). Светофор по факторам: `SeverityGreen` (#66D17A), `SeverityYellow` (#FFC857), `SeverityRed` (#FF5A5F).

**Светлая тема:** полная `LightColorScheme` с теми же смысловыми ролями на светлом фоне (#F4F6FF).

## Экраны

### Онбординг (OnboardingScreen)

Анкета пользователя: имя, возраст (Slider 18–90, опционально), тип метеочувствительности (`SingleChoiceSegmentedButtonRow`: лёгкая / средняя / сильная), хронические состояния (чипы). После завершения создаётся профиль в Room и приложение переходит на главный экран. Чувствительность влияет на множитель `personalPenalty` в `WellbeingCalculator`.

### Главный экран (DashboardScreen)

Отображает три карточки:
- **Индекс самочувствия** — крупное число «X из 100» с цветовым кодированием + подпись «X из 10 — короткий текст из `WellbeingLevel`» под ним. Под цифрами — `SeverityBadge`: трёхцветный светофор (зелёный/жёлтый/красный), классификация определяется `WellbeingCalculator.classifySeverity()`.
- **Текущая погода** — температура (с иконкой `Thermostat`, окрашенной по серьёзности ΔT), давление (`Compress`), влажность (`WaterDrop`), ветер (`Air`). Цвет иконок зависит от пороговых значений из `FactorThresholds.kt` (зелёный/жёлтый/красный).
- **Геомагнитная активность** — Kp-индекс с иконкой `Bolt`, текстовая интерпретация и подпись «вклад в индекс: −N баллов» (вычисляется из формулы), чтобы пользователь видел реальный вес Kp.

Реальные ΔP_6h и ΔT_24h берутся из `weather_history` через `WeatherRepository.getHistoricalPressure(6)` / `getHistoricalTemperature(24)`. Обновление данных — `PullToRefreshBox` (свайп вниз).

### Прогноз (ForecastScreen)

Сверху — карточка `WellbeingTimelineCard` с графиком индекса самочувствия по дням (Vico `LineCartesianLayer`, отображается при ≥2 днях). Если у пользователя есть личный прогноз ML, в график подставляется `personalIndices`.

Далее — список карточек на 5 дней. Каждая карточка содержит:

- Дату, диапазон температур, описание погоды.
- Строку с дополнительными метеопараметрами: средняя влажность (💧) и скорость ветра (💨) за день.
- Диапазон времени графика (например, «03:00 – 21:00») — справа от строки с параметрами.
- График температуры (Vico `LineCartesianLayer`) с плавными кубическими кривыми (`PointConnector.cubic`), градиентной заливкой под линией и нижней осью времени. При касании отображается маркер с температурой в точке. График скрывается, если за день доступен менее чем 2 слот (актуально для текущего дня в начале суток).
- Два значения индекса самочувствия:
  - **Общий** — рассчитан `WellbeingCalculator` по стандартной формуле.
  - **Личный** (бейдж «личный NN») — предсказание `WellbeingPredictor`, обученного на дневнике пользователя. Отображается только при наличии ≥10 записей с метеоданными.

### Дневник (DiaryListScreen / DiaryAddScreen)

Список записей дневника самочувствия с возможностью удаления свайпом (`SwipeToDismissBox`). FAB открывает экран добавления записи.

Карточка записи содержит цветную вертикальную полоску (акцент) слева — цвет соответствует уровню самочувствия (синий / зелёный / янтарный / оранжевый / красный). Иерархия элементов: уровень самочувствия (крупно, акцентным цветом) + дата (мелко, справа) → симптомы → заметка → метеоданные (минимальным шрифтом).

Экран добавления: выбор самочувствия (`FilterChip`), симптомов, произвольная заметка. При сохранении к записи прикрепляются текущие метеоданные из кэша.

Кнопка анализа триггеров в TopAppBar открывает `TriggersScreen`.

### Анализ триггеров (TriggersScreen)

Вычисляет корреляцию Пирсона между уровнем самочувствия из дневника и тремя факторами: атмосферное давление, температура, Kp-индекс.

Требует минимум 14 записей; при нехватке данных показывает прогресс-бар заполнения. Результаты отображаются в виде карточек с коэффициентом `r`, текстовой интерпретацией силы и направления связи, и визуальной шкалой.

### Советы (RecommendationsScreen)

Список персональных советов из `assets/recommendations.json`. Фильтрация по текущим погодным условиям и профилю пользователя: показываются только те советы, условия которых совпадают с текущим состоянием.

Сверху списка — `DisclaimerBanner`: «Рекомендации носят информационный характер и не заменяют консультацию врача.»

Каждый совет содержит поля `category` (`activity` / `nutrition` / `lifestyle` / `medical`) и опциональный `source` (например, «Минздрав РФ»). Если `source` указан, он отображается в подвале карточки серым подписным шрифтом «Источник: …».

Карточки оформлены в едином стиле с дневником: фон `surfaceVariant`, левая акцентная полоска `secondary` (янтарный).

### Настройки (SettingsScreen)

Секции:
- **Уведомления** — главный тумблер; при включении раскрывается группа из 4 переключателей (скачки давления / магнитные бури / морозы / жара), которые управляют per-event фильтрацией в `WeatherSyncWorker`.
- **Внешний вид** — переключатель тёмной темы.
- **Единицы измерения** — выбор единицы давления: мм рт.ст. / гПа (`SingleChoiceSegmentedButtonRow`).
- **Местоположение** — текстовое поле «Город» (fallback) и кнопка «Определить автоматически». Кнопка запрашивает `ACCESS_FINE_LOCATION` через `rememberLauncherForActivityResult` и сохраняет координаты в профиль через `LocationRepository`.
- **Данные** — очистка дневника с подтверждающим `AlertDialog`.
- **О приложении** — пункты «Источники» (`SourcesScreen`) и «Конфиденциальность» (`PrivacyScreen`).

### Источники (SourcesScreen)

Карточный список авторитетных справочников: ВОЗ, Минздрав РФ, ГРЛС, NOAA SWPC, OpenWeatherMap — каждый с описанием и ссылкой.

### Конфиденциальность (PrivacyScreen)

Раскрывает политику данных: локальное хранение, состав запросов к OpenWeatherMap/NOAA, использование геолокации только по запросу, локальные уведомления, соответствие 152-ФЗ и GDPR. Кнопка «Удалить все данные» очищает дневник и сбрасывает профиль через `PrivacyViewModel.wipeAllData()`.

## Навигация

Нижняя панель `NavigationBar` с пятью вкладками: Главная, Прогноз, Дневник, Советы, Настройки. Скрывается на экранах онбординга, добавления записи и анализа триггеров.

Цвета бара заданы вручную: `containerColor = surface`, `tonalElevation = 0` (без M3-оверлея), индикатор активной вкладки — `primaryContainer`, активная иконка/текст — `primary`, неактивные — `onSurfaceVariant`.

Маршруты:

| Маршрут | Экран |
|---|---|
| onboarding | OnboardingScreen |
| dashboard | DashboardScreen |
| forecast | ForecastScreen |
| diary | DiaryListScreen |
| diary/add | DiaryAddScreen |
| diary/triggers | TriggersScreen |
| recommendations | RecommendationsScreen |
| settings | SettingsScreen |
| settings/sources | SourcesScreen |
| settings/privacy | PrivacyScreen |

## Доступность

Все интерактивные иконки снабжены атрибутом `contentDescription` для поддержки TalkBack. Цветовое кодирование дополняется текстовыми метками (цвет не является единственным носителем смысла).
